# 🏥 多機能服薬管理システム (Health-Log App) -- ※アプリ開発中でまだ完成していません。

## 📋 プロジェクト概要
複数の病院や診療科に通う患者が、バラバラに処方される薬を一括管理し、飲み合わせや通院スケジュールを可視化するためのWebアプリケーションです。

## 🛠 技術スタック
- **Backend**: PHP 8.5 / Laravel 13.4
- **Frontend**: Tailwind CSS (CDN)
- **Database**: MySQL (HeidiSQLにて管理)
- **Infrastructure**: Docker (Laravel Sail) / Ubuntu (WSL2)

## 🏗 データ構造（ER図の概念）
本システムは「いつ、どこで、何を」を正確に管理するため、以下の4階層リレーションを採用しています。
- **Hospitals (1)** ↔ **Departments (多)** : 1つの病院に複数の診療科
- **Departments (1)** ↔ **Prescriptions (多)** : 1つの科から発行される複数の処方箋（日付単位）
- **Prescriptions (1)** ↔ **Medicines (多)** : 1つの処方箋に含まれる複数の薬剤

## 🚀 主要な開発ログとトラブルシューティング
開発中に直面した技術的課題とその解決策の記録です。

### 1. Mass Assignment（一括割り当て）の保護
- **課題**: データの保存時に `MassAssignmentException` が発生。
- **解決**: Laravelのセキュリティ仕様を理解し、各Modelの `$fillable` プロパティに保存許可カラムを明示的に定義。
- **学び**: 意図しないカラムの書き換えを防止するセキュリティ設計の重要性を習得。

### 2. 階層化されたリレーションのルーティング
- **課題**: 薬の登録時に、どの処方箋に紐付けるかを判別できず404エラーが発生。
- **解決**: URLパラメータ（query string）を利用して `prescription_id` を画面間で引き継ぐ設計に変更。
- **学び**: ユーザーの操作コンテキスト（文脈）をURLで保持するステートレスな設計手法を実践。

### 3. N+1問題の回避と高速化
- **課題**: ダッシュボードで全データを表示する際、クエリ回数が膨大になる懸念。
- **解決**: `Eloquent` の `with(['department.hospital', 'medicines'])`（Eager Loading）を使用し、クエリ回数を最小限に抑制。
